通过ssh远程访问内网计算机
2012年03月13日 ⁄ linux ⁄ 评论关闭

背景:我在内网搭建了linux系统A,想在另外一台外网计算机B上对其进行控制,已知A是内网计算机,无公网IP,B有固定公网IP或者NAT转发也行,在这种情况下,可以通过强大的ssh服务来实现要求.
首先现在系统A上执行下面的指令
ssh -R 1234:localhost:22 -l root xxx.xxx.xxx.xxx

解释下:
这条指令是监听B系统的端口1234,使其所有数据转发到A系统的22端口上,xxx.xxx.xxx.xxx是B系统的公网IP,root为B系统的用户,也可以为其他的用户,不一定是root.
执行完之后就可以在B系统上通过
ssh -p 1234 username@localhost
来访问A系统了
username是B系统的用户.

这个只是一个示例,举一反三,只要A系统上开的服务都可以通过这个方法来访问,比如web,ftp,等等.
好处:
1,以前我都是通过vpn来连接内网计算机的,但是用了这个方法之后速度明显感觉快多了,这样两台计算机的数据传输不需要经过第三方转发,所以速度快多了.
2,安全,所有数据都是经过加密传输,不会被有心人检测到你的私人信息,比如用户名,密码之类.
